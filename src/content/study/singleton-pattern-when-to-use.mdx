싱글턴은 "인스턴스를 딱 하나만 만들어서 전역으로 공유하는" 패턴이에요.
개념은 단순한데, <strong>어디에나 쓸 수 있어 보인다는 점</strong>이 오히려
문제가 됩니다.

## 실제로 쓰는 상황

- 설정값(config) 객체 — 앱 전체에서 같은 설정을 참조해야 할 때
- 로그 기록기(logger) — 여러 곳에서 같은 로그 스트림에 써야 할 때
- 커넥션 풀 — 리소스가 무겁고, 여러 개 만들면 오히려 손해일 때

## 남발하면 안 되는 이유

전역 상태처럼 동작하기 때문에 테스트가 어려워져요. 테스트마다 상태가
공유되면서 이전 테스트의 결과가 다음 테스트에 영향을 주는 경우가 생깁니다.
"공유되면 편할 것 같다"는 이유만으로 쓰기 시작하면, 나중에 그 클래스에
의존하는 코드를 전부 리팩터링해야 할 수도 있어요.

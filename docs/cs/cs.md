# CS
- **추상클래스, 인터페이스**
  **추상클래스는** 선언부만 작성하고 구현부는 작성하지 않은 추상 메서드들을 가지고 있다. 상속을 통해 자식 클래스에서 완성하도록 유도하는 클래스라고 볼수 있다. 추상클래스의 경우 일반 메서드와 변수를 가질수 있습니다.

  **인터페이스는** 일종의 추상 클래스로 추상 메서드를 갖지만 일반 메서드와 멤버 변수를 가질 수 없다. 완전한 빈 껍데기이다. 

- **오버로딩, 오버라이딩**
  **오버로딩** : 함수 중복 정의. 함수에 매개변수를 다르게 사용하여 매개변수에 따라 다른 함수가 실행되는것
  **오버라이딩** : 상속을 받았을때 부모 클래스의 함수 대신 자식 클래스에서 함수를 재 정의 하여 사용하는것

- **해시테이블**
  Key와 value로 데이터를 저장하는 자료구조 중 하나로 빠르게 데이터를 검색할 수 있는 자료구조이다. 해시테이블은 각각의 Key값에 해시 함수를 적용해 배열의 고유한 index를 생성하고, 이 index를 사용해 값을 저장하거나 검색을 하게된다. 메모리를 사용하여 시간을 줄이는 기법인데 Key값으로 데이터를 찾을때 보통 해시 함수 한번만 수행하면 되므로 매우 빠르게 데이터를 조회/삭제 할 수 있다. 해시테이블의 평균 시간복잡도는 O(1)이다.

- **메모리 영역**
  메모리 영역은 크게 4가지로 나뉩니다. 각각 code / data / stack / heap 영역 이라 부른다.
  **code 영역** : 실행할 프로그램의 코드가 저장되는 영역. CPU는 code 영역에서 저장된 명령어를 하나씩 가져가 처리합니다.
  **data 영역** : 프로그램의 전역 변수와 정적 변수가 저장되는 영역. data 영역은 프로그램의 시작과 함께 할당되며 프로그램이 종료되면 소멸합니다.
  **stack 영역** : 함수의 호출과 관계되는 지역변수와 매개변수가 저장되는 영역. stack 영역은 함수의 호출과 함께 할당되면, 함수의 호출이 완료되면 소멸합니다. 
  **heap 영역** : 사용자가 직접 관리할수있는 메모리 영역. 메모리 공간이 동적으로 할당되고 해제됩니다. heap 영역은 메모리 관리에 주의를 기울여야 하며 메모리 단편화가 발생할 수 있다.

- **프로세스와 스레드의 차이**
  **프로세스** : 운영체제로 부터 시스템 자원을 할당받는 작업의 단위. 프로세스는 각각의 독립된 메모리 영역(code / data / stack / heap) 을 할당 받는다
  **스레드** : 프로세스 내에서 실행되는 여러 흐름의 단위. 기본적으로는 프로세스당 1개의 스레드를 가지고 있다.

- **Context switching**
  CPU  에서 여러 프로세스를 돌아가면서 작업을 처리하는데 이 과정을 context swiching 이라 한다. 동작중인 프로세스가 대기하면서 해당 프로세스의 상태를 보관하고, 다시 실행시 복구하게 되는데 이 때 필요한 비용을 말한다.

- **뮤텍스와 세마포어**

  **뮤텍스** : 공유된 자원에 여러 쓰레드가 접근하는 것을 막는것
  **세마포어** : 공유된 자원에 여러 프로세스가 접근하는 것을 막는것
